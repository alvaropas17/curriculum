Caso de Proyecto · IA Educativa

Aprender no debería depender de repetir preguntas al azar.

LudoScript es una plataforma educativa que utiliza IA y temario estructurado para crear experiencias de práctica con quizzes, flashcards, seguimiento de progreso y revisión de errores. La demo pública utiliza contenido precargado para garantizar una experiencia estable.

Vue 3 Express PostgreSQL Supabase Gemini API
ludoscript.vercel.app

El Problema

Muchas plataformas miden el acierto, pero no la comprensión real ni el porqué de los errores.

visibility_off

Ilusión de aprendizaje

Acertar preguntas puede dar una falsa sensación de dominio mientras persisten lagunas críticas en la base del conocimiento.

person_search

Falta de personalización

El enfoque de "talla única" ignora que cada estudiante tropieza en diferentes conceptos y requiere ritmos distintos.

navigation

Sin dirección clara

Tras una sesión de práctica, el estudiante a menudo no sabe qué repasar específicamente o cuál es el siguiente paso lógico.

La Hipótesis

"El valor no está en generar preguntas con IA, sino en usar esas preguntas para diagnosticar el estado del alumno y guiarle con intención."

Cómo Funciona

Un flujo de aprendizaje guiado a partir de temario preparado para practicar, medir progreso y revisar errores con claridad.

01

Selección de Temario

El alumno elige un temario precargado o un módulo de ejemplo para empezar a practicar sobre contenido ya estructurado.

02

Actividades Asociadas

La plataforma muestra preguntas, flashcards y conceptos asociados al contenido para que el aprendizaje tenga un recorrido claro.

03

Práctica y Seguimiento

El alumno interactúa con las actividades y el sistema registra aciertos, errores y progreso para hacer visible su evolución.

04

Revisión de Puntos Débiles

La plataforma ayuda a identificar qué partes conviene repasar a partir del historial de práctica y los errores cometidos.

Funcionalidad experimental

La subida de documentos propios está planteada para permitir que el usuario genere actividades a partir de sus materiales. Actualmente se mantiene como función experimental mientras se mejora la estabilidad del procesamiento.

Retos y Aprendizajes

Algunas decisiones técnicas del proyecto surgieron al equilibrar coste, fiabilidad y una experiencia de aprendizaje clara.

01

Optimización de Tokens

Controlar el coste de la IA fue crucial. Implementé flujos que maximizan el valor de cada prompt, evitando llamadas redundantes y procesando datos en lotes eficientes.

02

Contenido Estructurado

Preparar temario y actividades coherentes fue clave para que la demo mostrara el flujo principal sin depender de procesamiento experimental de documentos.

03

Enfoque a Producto

Aprendí que una funcionalidad técnica no sirve de nada sin una UX que la respalde. Diseñé una experiencia orientada al usuario y no solo a la tecnología.

04

IA Aplicada

La integración de modelos generativos requiere un manejo de errores robusto. Aprendí a validar salidas de IA para reducir errores, estructurar mejor las respuestas y mejorar la fiabilidad del contenido generado.
