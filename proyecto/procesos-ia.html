Un archivo que da a la IA mejor contexto para decidir.

origen

La idea

La calidad de una respuesta no depende solo del modelo: también depende del contexto, las restricciones y los criterios con los que trabaja. Un buen harness —el conjunto de guías que acompaña al agente— reduce ambigüedades y hace el proceso más predecible.

OpenCode permite cargar un archivo global (karpathy.md) como contexto en todos los proyectos. Así, los principios de trabajo no se pierden al cambiar de repositorio: el agente parte de una base común y adapto después las instrucciones específicas de cada caso.

problema

El contexto se reinicia

Sin una base común, cada proyecto empieza con las mismas aclaraciones: cómo analizar, qué priorizar y hasta dónde cambiar.

solución

Archivo global

Un solo fichero en ~/.config/opencode/ que opencode carga siempre. Las reglas están ahí desde el primer mensaje.

efecto

Consistencia

El agente no empieza de cero: conoce mis principios de ingeniería, mi estilo de trabajo y cómo exponer los trade-offs.

configuración

Dónde vive el archivo

opencode busca instrucciones globales en una ruta específica del sistema. Basta con colocar el archivo ahí y todos los proyectos lo heredan.

1. Crear el archivo

Escribes karpathy.md con las guías de comportamiento que quieras: cómo abordar problemas, qué priorizar, cómo documentar.

2. Colocarlo en la ruta global

Lo guardo en ~/.config/opencode/ y OpenCode lo carga automáticamente al iniciar cualquier proyecto.

3. Un punto de partida común

Cada conversación empieza con estas guías presentes, sin repetir configuración ni copiar archivos entre proyectos.

contenido

Qué contiene karpathy.md

Cuatro principios que reducen errores y mejoran la calidad del código generado por IA.

  1. Think Before Coding

    qué dice No asumas ni ocultes la incertidumbre. Expón los trade-offs antes de implementar.

    para qué sirve Evita que la IA se lance a escribir código sin entender el contexto o sin preguntar cuando algo es ambiguo.

  2. Simplicity First

    qué dice El mínimo código que resuelve el problema. Nada especulativo, nada abstracto para un solo uso.

    para qué sirve La IA tiende a sobreingenierizar. Este principio la frena y la obliga a preguntarse si está añadiendo complejidad innecesaria.

  3. Surgical Changes

    qué dice Cambia solo lo necesario. No “mejores” código adyacente ni refactorices lo que no está roto.

    para qué sirveEvita reescrituras que no aportan valor y limita cada intervención a lo estrictamente solicitado.

  4. Goal-Driven Execution

    qué dice Define criterios de éxito. Itera hasta verificarlos.

    para qué sirve Transforma instrucciones vagas en un bucle de trabajo comprobable. La IA sabe cuándo ha terminado.

resultado

Menos fricción, mejores decisiones

Las dudas se hacen visibles

Cuando algo no está claro, el agente lo dice en lugar de asumir. Eso evita iteraciones desperdiciadas y código que no pedí.

Menos sobreingeniería

El archivo frena las abstracciones innecesarias. El resultado es código más legible y fácil de mantener.

Consistencia entre proyectos

Da igual si empiezo un script pequeño o una app completa: el agente parte con la misma base de criterios.

Iteraciones con cierre verificable

Al tener objetivos claros desde el principio, cada ciclo de feedback produce cambios concretos y medibles.

el archivo

Un harness persistente para trabajar con mejor criterio.

El archivo global convierte una preferencia de trabajo en una base reutilizable: se configura una vez, acompaña a cada proyecto y deja espacio para el contexto específico de cada tarea.
